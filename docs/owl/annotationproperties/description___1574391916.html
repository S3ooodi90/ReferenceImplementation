Annotation Property: description

Usage (19)

  • CMC description "Core Model Component
    A composable model contained in a reference model. A CMC represents a specific core type of component that further contains elements with base datatypes and/or other CMCs to define its structure."     @en-us 
  • CMS description "Core Model Symbol
    A CMS represents a CMC in instance data. In practice, it is usually substituted for by a Replaceable Model Symbol (RMS). This substitution is due to the fact that constraints are expressed in a Replaceable Model Component (RMC) which is then represented by a RMS."     @en-us 
  • DM description "Data Model
    A set of selected RMCs that are constraints on the RM components (CMCs) in order to represent a domain concept. In the implementation language there may be additional syntactic conventions required. Caution: Not to be confused with Data Instance."     @en-us 
  • DataInstance description "A set of data items that reports via isInstanceOf property that it conforms to a DM. In this state it has not been tested for validation." @en-us 
  • DataInstanceError description "Subclass of DataInstance. A set of data items that DOES NOT conform to the DM it represents OR it contains invalid data and does not contain one or more children of an Exception. If there are no noted Exceptions in the Data Instance then it should be considered suspect and discarded." @en-us 
  • DataInstanceInvalid description "Subclass of DataInstance. A set of data items that conforms to a DM to represent an instance of that concept AND at least some of the data values are NOT valid according to the DM constraints. An Invalid Data Instance must contain one or more children of an Exception." @en-us 
  • DataInstanceValid description "Subclass of DataInstance. A set of data items that conforms to a DM to represent an instance of that concept AND the all of the data values are valid according to the DM constraints." @en-us 
  • Exception description "Exceptions indicate that a data instance may be invalid when validated against its model. However, the exceptional value may be explainable and provide valuable contextual information. A specific Excpetion subclass will provide that additional information." @en 
  • RM description "Reference Model

    A set of components called Core Concept Models (CMCs) that provide structural integrity for a domain concept. Some CMCs are mandatory in DMs and some are optional. Optionality is defined in each RM implementation."     @en-us 
  • RMC description "Reusable Model Component

    The name given to a CMC that has been constrained for use in a DM. Through the constraints, a RMC defines a single concept based on syntactic data constraints as well as specified semantics. It is reusable because it can be reused in multiple DMs."     @en-us 
  • RMS description "Reusable Model Symbol

    Represents a RMC in instance data. Can be considered as a data container for the components of a RMC. All RMSs are contained in a DataInstance when in transit."     @en-us 
  • S3Model description "The root concept. The abstract idea of S3Model. All of the S3Model classes are subclasses of this class." @en-us 
  • basedOnRM description "Relates a DM to one RM." @en-us 
  • containsCMC description "Defines the relation that a RM contains a specific CMC." @en-us 
  • containsRMC description "Defines the relation that a DM contains a specific RMC." @en-us 
  • hasException description "Relates an Exception subclass to an invalid data instance." @en 
  • isCMSOf description "Relates a symbol (CMS) to a model (CMC)." @en-us 
  • isInstanceOf description "Relates a DataInstance to a Data Model (DM)." @en-us 
  • isRMSOf description "Relates a symbol (RMS) to a model (RMC)." @en-us
